Getting Started with NVDLA S/W and VP

This is a tutorial for helping get started with NVDLA S/W and Virtual Platform. NVDLA is introduced by NVIDIA as an eco-system contains both SW and HW for doing CNN inference.

The Software Layer contains two parts: The first is compiler and the other part contains user space driver (UMD) and kernel space driver (KMD). To get started with the S/W we can first use git clone for the code from the repo: https://github.com/nvdla/sw

In the S/W repo, there are four main directories: umd, kmd, regression and prebuilt. In the prebuilt directory, you will find nvdla_compiler and nvdla_runtime (test S/W leverages on UMD). These are two prebuilt test binary for you can test on the NVDLA/Virtual Platform and in this tutorial we will focus on these two prebuilt binaries.

For more information for the compiler, umd, kmd one can following the tutorial: http://nvdla.org/sw/contents.html

To use nvdla_compiler, you can use the following command to generate the Loadable which is an intermediate product to be used in nvdla_runtime.

./nvdla_compiler --prototxt caffemodel.prototxt --caffemodel caffe.caffemodel

Notice that nvdla_compiler now only supports for caffe model, and it cannot support for tensorflow. The following is an example that I have tried can be converted to Loadable successfully.

AlexNet: https://github.com/BVLC/caffe/tree/master/models/bvlc_alexnet

After running nvdla_compiler, you will find that under the current directory, there will be one output: basic.nvdla and this is the output we need to pass to the nvdla_runtime to run the inference.

To test the inference, you will need to setup the virtual platform by following this tutorial: http://nvdla.org/vp.html. After you login this Virtual Simulator, you can use the prebuilt nvdla_runtime to run the inference. Let’s try with the loadables and images in the regression folder:

# First: you need to start the virtual platform docker
docker run -it -v /home:/home nvdla/vp# Second: go to the mount directory and copy the s/w folder we just pull beforecd /usr/local/nvdla
cp -r /path/to/sw/dir /usr/local/nvdla# Third: Start virtual simulator./aarch64_toplevel -c aarch64_nvdla.lua# Fourth: Login and mount the directorya. Login as root and password: nvdla
b. mount -t 9p -o trans=virtio r /mnt
c. cd /mnt# Fifth: Run Inference with the images and Loadable in regression folder
./nvdla_runtime --loadable ./sw/prebuilt/linux/flatbufs/kmd/NN/NN_L0_0_fbuf --image ./sw/regression/images/digits/seven.pgm --rawdump

After the fifth step is done, you will find the directory contains a new output output.dimg, and you can use cat output.dimg to see the inference result. For the above example I got the following result and you can find that number “7” has the largest probability :)

2.6226e-06 0.0133286 2.27094e-05 0.00288963 4.82798e-05 7.83801e-05 2.38419e-07 0.983398 3.57628e-07 0.000276089
