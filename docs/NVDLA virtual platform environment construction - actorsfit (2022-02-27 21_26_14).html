NVDLA virtual platform environment construction

Official link http://nvdla.org/vp.html 

Download the VP source code

# sudo apt-get install git//Install git first

$ git clone https://github.com/nvdla/vp.git 

$ cd vp  

$ git submodule update --init --recursive//Update the submodule file

Installation dependencies

$ sudo apt-get update $ sudo apt-get install g++ cmake libboost-dev python-dev libglib2.0-dev libpixman-1-dev liblua5.2-dev swig libcap-dev libattr1-d

Install systemC

Note: need to specify systemc2.3.0 version, higher version is not compatible

$ wget -O systemc-2.3.0a.tar.gz https://www.accellera.org/images/downloads/standards/systemc/systemc-2.3.0a.tar.gz

//Using https link, http cannot access $ tar -xzvf systemc-2.3.0a.tar.gz $ cd systemc-2.3.0a $ sudo mkdir -p/usr/local/systemc-2.3.0/$ mkdir objdir $ cd objdir $ ../configure --prefix=/usr/local/systemc-2.3.0 $ make $ sudo make install

Install perl package (YML and Tee)

$ wget -O YAML-1.24.tar.gz https://search.cpan.org/CPAN/authors/id/T/TI/TINITA/YAML-1.24.tar.gz

$ tar -xzvf YAML-1.24.tar.gz $ cd YAML-1.24 $ perl Makefile.PL

$ make

$ sudo make install

$ wget -O IO-Tee-0.65.tar.gz https://search.cpan.org/CPAN/authors/id/N/NE/NEILB/IO-Tee-0.65.tar.gz

$ tar -xzvf IO-Tee-0.65.tar.gz $ cd IO-Tee-0.65

$ perl Makefile.PL

$ make

$ sudo make install

Build NVDLA CMOD

$ git clone https://github.com/nvdla/hw.git

$ cd hw

$ make

The following prompt appears at the beginning of make, you need to specify the path of the construction environment

View Image

The dependent environment is as follows

  • Java - jdk1.7
  • Perl - perl-5.10
    • XML::Simple
    • Capture::Tiny
  • CPP - gcc-4.9.3
  • Python - python2.6
  • SystemC - systemc-2.3.0
  • (for Verilator builds) Verilator - Verilator 3.912
  • (for Verilator builds) clang - clang 3.4

You can modify the default path in the MakeFile file before make. Paths such as cpp can be searched with whereis cpp. The general path is/ust/bin/cpp

$ tools/bin/tmake -build cmod_top

The successful construction is as follows

View Image

The generated files are in the directory hw/outdir/<project>/cmod/release.

Build and install the emulator

# cd vp//Switch to vp warehouse

$ cmake -DCMAKE_INSTALL_PREFIX=build -DSYSTEMC_PREFIX=/usr/local/systemc-2.3.0/-DNVDLA_HW_PREFIX=/home/sunqiliang/VP/hw -DNVDLA_HW_PROJECT=sun_test

//The installation path of the DCMAKE_INSTALL_PREFIX emulator systemc prefix is ​​the path of the installed systemc nvdla_hw prefix is ​​the NVDLA HW warehouse path nvdla_hw project is the HW project name, the project name in the previous step is sun_test

Cross compile a linux kernel

wget https://buildroot.org/downloads/buildroot-2017.11.tar.gz

tar -zxvf buildroot-2017.11.tar.gz

cd buildroot-2017.11/

 sudo apt-get install libncurses5-dev//will be used when make menuconfig

$ make qemu_aarch64_virt_defconfig

$ make menuconfig
